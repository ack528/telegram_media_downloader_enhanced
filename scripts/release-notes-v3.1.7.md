# v3.1.7 更新说明

本次版本重点修复启动后卡住、不开始下载的问题。

## 修复内容

- 修复读取 Telegram 历史消息时请求可能长期不返回，导致任务停在启动阶段、不会进入下载队列的问题。
- 新增历史消息读取超时与重试机制，默认每次请求等待 60 秒，最多重试 3 次。
- 历史读取最终失败时会记录明确日志，包含任务 ID、会话 ID 和 last_read_message_id，便于快速定位。
- 没有发现新可下载消息时会写入明确日志，避免误判为软件卡死。
- 修复机器人状态消息读取 Clash 下载速度时可能阻塞事件循环的问题；Clash `/traffic` 现在按流式接口在后台刷新首条速度数据，并带独立超时保护。
- 降低机器人状态消息编辑频率，减少 Telegram FloodWait 导致长时间不更新通知的问题。
- Python 窗口新增启动完成、机器人等待命令、收到下载任务、发现首个媒体并开始入队等简洁提示。
- 记录机器人状态消息创建结果，便于排查 config/恢复任务没有机器人通知的问题。
- 新增 `scan_prefetch_limit`，默认按下载线程数限制扫描预取数量，避免 1-1000 这类任务先轮询大量媒体 ID 再下载；完成一个下载后继续向后扫描。
- 修复已完成文件存在时的临时文件清理逻辑；异常退出后如果部分最终文件和 `.temp` 断点文件同时存在，会保留更大的有效断点继续下载，避免重复下载或丢失进度。

## 配置新增

```yaml
history_fetch_timeout: 60
history_fetch_retries: 3
scan_prefetch_limit: 5
```

旧配置文件不添加也可以运行，程序会默认使用 `max_download_task` 作为预取上限；如需恢复旧版无限预取，可设置 `scan_prefetch_limit: 0`。
